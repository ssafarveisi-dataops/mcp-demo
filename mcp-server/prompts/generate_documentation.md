# Generate Documentation Instructions

Use this prompt when the user asks to **generate documentation from code or files**.

## Instructions
- Read the content of the specified code file(s).
- Extract:
  - Function/class definitions
  - Purpose and functionality of each function/class
  - Input arguments and expected output (if clear)
  - Any docstrings or comments
- Produce a structured documentation format (Markdown preferred):
  - File name / path
  - Function/Class name
  - Description
  - Parameters and return values
- Ensure clarity and correctness; do not invent functionality.
